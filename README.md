# Hard-Duck — інструмент захисту робочих станцій Windows

**Версія 1.0.15** | Внутрішній інструмент керування кінцевими точками

Hard-Duck — десктопний WPF-застосунок та автономний PowerShell-агент для захисту
робочих станцій Windows 10/11. Автоматизує ротацію облікових даних локального
адміністратора (легковагова альтернатива Microsoft LAPS для середовищ без Entra ID),
увімкнення BitLocker, блокування USB-накопичувачів, налаштування Windows LAPS та
безпечну передачу ключів відновлення BitLocker до Infisical Cloud.

---

## Архітектура

```
┌──────────────────┐     download + run    ┌──────────────────────────────┐
│   Hard-Duck.exe  │ ────────────────────> │       hard-duck.ps1          │
│  (WPF .NET 8)    │   HttpClient → %TEMP% │                              │
│                  │   Process.Start       │  1. Збір системної інформації│
│ ┌──────────────┐ │                      │  2. Ротація пароля адміна    │
│ │  UI Toggles  │ │   -EnableHardDuckAdmin│  3. Увімкнення BitLocker      │
│ │  + FAB Run   │ │   -EnableSecureBoot  │  4. Отримання recovery-ключа  │
│ └──────────────┘ │   -DisableUSB ...    │  5. Формування секрету        │
└──────────────────┘                      └───────────┬──────────────────┘
                                                      │  HTTPS + Bearer token
                                                      ▼
                                          ┌──────────────────────────────┐
                                          │     Infisical Cloud API       │
                                          │  POST /api/v3/secrets/raw/    │
                                          │       coati-secret-storage-   │
                                          │       qu-pc/dev               │
                                          │                              │
                                          │  Secret: DEVICE_<Serial>      │
                                          │  Value:  { JSON-дані }        │
                                          └──────────────────────────────┘
```

Hard-Duck використовує тонкоклієнтну архітектуру: WPF-застосунок завантажує
`hard-duck.ps1` з публічного GitHub-репозиторію під час виконання, передає
прапорці з UI як аргументи командного рядка, і виконує скрипт через
`powershell.exe -File`. Усі зібрані секрети передаються через HTTPS до
Infisical Cloud. Жодних локальних секретів на диску не залишається.

## Можливості

| Можливість | Опис |
|---|---|
| **Ротація пароля адміністратора** | Генерує криптостійкий 12-значний пароль і встановлює його на вбудований обліковий запис Administrator (знаходиться за SID `*-500` для кросс-мовної сумісності). Якщо обліковий запис вимкнено — вмикає його. |
| **Увімкнення BitLocker** | Перевіряє статус диска C:. Якщо `FullyDecrypted`, вмикає BitLocker із протектором TPM-only (`XtsAes256`). Чекає до 120 секунд на появу протектора `RecoveryPassword`. |
| **Ескроу recovery-ключа** | Отримує 48-значний пароль відновлення BitLocker і пакує його разом з ідентифікаційними даними машини в JSON. |
| **Захищене хмарне сховище** | Відправляє дані до Infisical Cloud як секрет із назвою `DEVICE_<СерійнийНомер>` у середовищі `dev`. Уся комунікація — через HTTPS з Bearer-токеном. |
| **Secure Boot** | Перевіряє стан Secure Boot в UEFI. |
| **USB-накопичувачі** | Блокує USB-накопичувачі через вимкнення драйвера USBSTOR. |
| **Windows LAPS** | Встановлює та налаштовує Windows LAPS (потребує Entra ID). |
| **BIOS-пароль** | Перевіряє наявність пароля BIOS/UEFI через Lenovo WMI. |
| **Непомітне виконання** | PowerShell викликається без вікна, без взаємодії з користувачем. |
| **Мовна незалежність** | Обліковий запис адміністратора шукається за SID (`*-500`), а не за назвою. Працює на будь-якій локалізації Windows. |

## Інструкція з розгортання

### Передумови

- **Windows 10/11** Pro, Enterprise або Education (BitLocker недоступний у Home).
- **TPM 2.0** присутній і готовий.
- **Обліковий запис Infisical Cloud** із доступом до проєкту `coati-secret-storage-qu-pc`.
- **Service Token** із правом запису в цільове середовище (`dev`).
- **Права локального адміністратора** (скрипт вимагає `#Requires -RunAsAdministrator`).

### Розгортання через WPF-застосунок

1. Завантажте останній реліз `Hard-Duck.exe` з [GitHub Releases](https://github.com/Homka13/Hard-Duck/releases).
2. Запустіть `Hard-Duck.exe` від імені адміністратора.
3. Позначте потрібні прапорці (Secure Boot, USB, BitLocker PIN, LAPS, Hard-Duck Admin).
4. Натисніть кнопку **🛡️** (FAB) для запуску процесу захисту.

Застосунок автоматично завантажить `hard-duck.ps1` з GitHub і виконає його
з переданими прапорцями.

### Автономний запуск PowerShell-скрипта

Для запуску без WPF-застосунку:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\Path\To\hard-duck.ps1" -EnableHardDuckAdmin
```

## Зауваги щодо безпеки

- **Захист Service Token:** Infisical Service Token, вбудований у `hard-duck.ps1`, має
  право запису в проєкт `coati-secret-storage-qu-pc`. Поводьтеся з файлом скрипта
  так само обережно, як із будь-яким артефактом, що містить облікові дані.
- **Безпека передавання:** Усі виклики API використовують HTTPS із TLS 1.2+. Токен
  передається в заголовку `Authorization: Bearer` — ніколи в URL чи рядку запиту.
- **Відсутність локального збереження:** Згенерований пароль адміністратора та ключ
  відновлення існують лише в пам'яті та у відповіді Infisical. Вони ніколи не
  записуються на диск, у журнали подій чи реєстр цим скриптом.
- **Контекст виконання:** Скрипт потребує прав адміністратора. Запускайте його лише
  на довірених, керованих машинах.
- **Увімкнення облікового запису:** Якщо вбудований обліковий запис Administrator
  було раніше вимкнено, цей скрипт увімкне його. Перевірте цю поведінку на
  відповідність політиці безпеки вашої організації.

## Пов'язані інструменти

WPF-застосунок **Hard-Duck** виконує повний цикл захисту робочої станції з
графічним інтерфейсом у вигляді чекліста:

| Етап | Опис |
|---|---|
| Secure Boot | Перевірка; зупинка, якщо вимкнено |
| TPM 2.0 | Перевірка готовності |
| Entra ID join | Перевірка повного приєднання |
| BitLocker | TPM-only або TPM+PIN із діалогом PIN |
| Recovery-ключ | Бекап до Entra ID |
| Гібернація | Примусово після 10 хв бездіяльності |
| USB-накопичувачі | Опційне блокування через вимкнення драйверів |
| BIOS-пароль | Перевірка стану через Lenovo WMI |
| Windows LAPS | Опційний LAPS на базі Entra ID |
| **Hard-Duck** | Ротація пароля адміністратора + ескроу ключа → Infisical |
| Права адміністратора | Інтерактивне зняття прав у щоденного користувача |

### Збірка Hard-Duck

Потрібен [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0):

```powershell
cd src
dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

Результат: `src\bin\Release\net8.0-windows\win-x64\publish\Hard-Duck.exe`

Застосунок використовує тонкоклієнтну архітектуру: завантажує `hard-duck.ps1`
з GitHub під час виконання і передає прапорці з UI як аргументи командного рядка.
